# 规则：交互标注

**优先级：** 中等 (MEDIUM)
**类别：** 交互标注

## 描述

交互标注为原型添加行为说明，使开发者和设计师能够理解原型的动态行为。清晰的标注减少沟通成本，确保实现与设计意图一致。

## 标注类型

### 页面级标注

**标注内容：**
- 页面名称和ID
- 页面入口（从哪里进入）
- 权限要求
- 页面状态说明

**标注格式：**
```
【页面信息】
页面名称：用户列表
页面ID：user-list
入口：主导航 > 用户管理
权限：管理员、运营人员
状态：正常/空状态/加载中
```

### 组件级标注

**标注内容：**
- 组件名称
- 数据来源
- 交互行为
- 状态变化

**标注格式：**
```
【搜索按钮】
类型：主按钮
行为：点击触发搜索
联动：刷新下方表格数据
加载：点击后显示loading状态
```

### 流程级标注

**标注内容：**
- 流程名称
- 起点和终点
- 决策分支
- 异常处理

**标注格式：**
```
【删除流程】
起点：点击删除按钮
分支1：确认删除 → 调用接口 → 刷新列表
分支2：取消删除 → 关闭弹窗
异常：删除失败 → 显示错误提示
```

## 交互连接线

### 连接线类型

| 类型 | 线条样式 | 含义 |
|-----|---------|------|
| 实线箭头 | ──────→ | 页面跳转 |
| 虚线箭头 | - - - - → | 弹窗打开 |
| 双向箭头 | ←──────→ | 标签切换 |
| 返回线 | ←────── | 返回上一页 |

### 连接标注写法

```
跳转标注：
[按钮A] ──「点击跳转」──→ [页面B]

弹窗标注：
[按钮A] ──「点击打开弹窗」──→ (弹窗B)

条件分支：
[提交按钮] ──「成功」──→ [成功页]
         └──「失败」──→ [错误提示]
```

## 标注符号系统

### 状态符号

| 符号 | 含义 | 使用场景 |
|-----|------|---------|
| 🔵 | 默认状态 | 初始状态标注 |
| 🟢 | 成功状态 | 完成状态标注 |
| 🟡 | 警告状态 | 警告提示标注 |
| 🔴 | 错误状态 | 错误提示标注 |
| ⚪ | 禁用状态 | 不可用状态标注 |
| ⏳ | 加载状态 | 处理中状态标注 |

### 操作符号

| 符号 | 含义 | 使用场景 |
|-----|------|---------|
| 👆 | 点击 | 点击触发 |
| ⌨️ | 输入 | 键盘输入 |
| 🖱️ | 悬停 | 鼠标悬停 |
| 📱 | 触摸 | 移动端触摸 |
| ⬆️ | 提交 | 表单提交 |
| 🔄 | 刷新 | 数据刷新 |

### 数据符号

| 符号 | 含义 | 使用场景 |
|-----|------|---------|
| 📥 | 接收数据 | 接口返回 |
| 📤 | 发送数据 | 接口请求 |
| 💾 | 本地存储 | 缓存数据 |
| 🔗 | 关联数据 | 数据依赖 |

## 常见交互标注模板

### 按钮交互标注

```
【保存按钮】
├── 位置：表单底部右侧
├── 样式：主按钮
├── 触发：点击
├── 校验：
│   ├── 必填字段非空 → 通过
│   └── 必填字段为空 → 显示错误提示
├── 行为：
│   ├── 校验通过 → 显示loading → 调用接口
│   │   ├── 接口成功 → 提示「保存成功」→ 返回列表
│   │   └── 接口失败 → 提示错误信息 → 保留当前页
│   └── 校验失败 → 滚动到第一个错误字段
└── 状态：
    ├── 正常：蓝色填充
    ├── 悬停：深蓝色
    ├── 加载：按钮内loading + 禁用
    └── 禁用：灰色填充
```

### 表格交互标注

```
【数据表格】
├── 数据来源：/api/users/list
├── 默认排序：创建时间倒序
├── 刷新触发：搜索、筛选、分页、操作完成
├── 行为：
│   ├── 点击行：跳转详情页
│   ├── 勾选行：启用批量操作按钮
│   └── 排序：点击表头切换排序
├── 操作列：
│   ├── 查看 → 跳转详情页
│   ├── 编辑 → 跳转编辑页
│   └── 删除 → 弹窗确认 → 刷新列表
└── 状态：
    ├── 正常：显示数据
    ├── 空状态：显示空状态图+文字
    ├── 加载：显示骨架屏
    └── 错误：显示错误提示+重试按钮
```

### 表单交互标注

```
【用户表单】
├── 提交方式：POST /api/users
├── 字段：
│   ├── 用户名
│   │   ├── 类型：文本输入
│   │   ├── 必填：是
│   │   ├── 校验：3-20字符，仅字母数字
│   │   └── 错误提示：「用户名为3-20位字母或数字」
│   ├── 手机号
│   │   ├── 类型：文本输入
│   │   ├── 必填：是
│   │   ├── 校验：11位手机号格式
│   │   └── 错误提示：「请输入正确的手机号」
│   └── 角色
│       ├── 类型：下拉选择
│       ├── 必填：是
│       ├── 选项：管理员/运营/普通用户
│       └── 默认：普通用户
├── 实时校验：失焦时触发
├── 提交：
│   ├── 按钮文字：「创建用户」
│   └── 成功后：返回列表页
└── 取消：
    ├── 按钮文字：「取消」
    └── 行为：返回列表页（如有修改则确认）
```

## 弹窗/模态框标注

```
【删除确认弹窗】
├── 触发：点击删除按钮
├── 类型：模态弹窗
├── 尺寸：400px宽
├── 内容：
│   ├── 标题：「确认删除」
│   ├── 正文：「确定要删除该用户吗？此操作不可恢复。」
│   └── 底部按钮：
│       ├── 取消（次按钮）→ 关闭弹窗
│       └── 确认删除（危险按钮）→ 执行删除
├── 交互：
│   ├── 点击遮罩：关闭弹窗
│   ├── 按ESC：关闭弹窗
│   └── 删除中：按钮显示loading
└── 结果：
    ├── 成功：关闭弹窗 → 提示「删除成功」→ 刷新列表
    └── 失败：关闭弹窗 → 提示错误信息
```

## 标注呈现方式

### 原型内标注

- 使用标注层，不影响原型主体
- 标注框使用统一颜色（建议橙色）
- 标注编号从上到下、从左到右排列

### 侧边注释

```
页面原型              │ 标注说明
─────────────────────┼──────────────────────
┌──────────────┐     │ 1. 搜索按钮
│ [搜索]       │←───│    - 点击触发搜索
│              │     │    - 支持回车触发
│ 列表数据...   │     │ 2. 数据表格
│              │     │    - 分页加载
└──────────────┘     │    - 每页10条
```

## 常见错误

- 标注过于简单，缺少关键信息
- 标注与原型分离，对照困难
- 忽略异常状态和边界情况
- 未标注数据来源和校验规则
- 交互流程描述不完整
- 使用模糊术语（如「跳转」未指明目标）
