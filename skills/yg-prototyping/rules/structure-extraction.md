# 规则：结构提取

**优先级：** 关键 (CRITICAL)
**类别：** 结构提取

## 描述

结构提取将解析后的需求文档转换为清晰的页面层级结构、导航关系和元素清单，是连接需求分析与原型设计的桥梁。

## 页面层级结构

### 层级定义原则

**三级结构模型：**

```
应用层级 (Application)
├── 模块层级 (Module)
│   ├── 页面层级 (Page)
│   │   ├── 区块层级 (Section)
│   │   │   └── 组件层级 (Component)
```

**层级划分规则：**
- 应用层级：整个产品/系统
- 模块层级：功能相关的页面集合
- 页面层级：一个完整的界面
- 区块层级：页面内的功能分区
- 组件层级：最小UI单元

### 页面命名规范

| 类型 | 命名格式 | 示例 |
|-----|---------|------|
| 列表页 | {实体}列表 | 用户列表、订单列表 |
| 详情页 | {实体}详情 | 商品详情、订单详情 |
| 表单页 | {动作}{实体} | 新增用户、编辑商品 |
| 设置页 | {功能}设置 | 账户设置、权限设置 |
| 仪表盘 | {模块}仪表盘/概览 | 数据概览、销售仪表盘 |

### 层级提取示例

**输入：需求文档功能清单**
```markdown
1. 用户管理
   - 用户列表查询
   - 用户详情查看
   - 新增用户
   - 编辑用户信息
2. 角色管理
   - 角色列表
   - 角色权限配置
```

**输出：页面层级结构**
```
应用：用户权限管理系统
├── 模块：用户管理
│   ├── 页面：用户列表
│   ├── 页面：用户详情
│   └── 页面：用户表单（新增/编辑共用）
└── 模块：角色管理
    ├── 页面：角色列表
    └── 页面：权限配置
```

## 导航关系梳理

### 导航类型识别

| 类型 | 特征 | 适用场景 |
|-----|------|---------|
| 主导航 | 固定入口，覆盖核心功能 | 全局可见 |
| 侧边导航 | 层级深，功能多 | 后台管理系统 |
| 底部导航 | 3-5个核心功能 | 移动端应用 |
| 面包屑 | 多层级嵌套 | 内容层级深的页面 |
| 标签导航 | 同级内容切换 | 并列内容展示 |

### 页面关系提取

**从需求中提取导航：**

```markdown
原文：「用户列表页点击详情按钮跳转到用户详情页」
提取：用户列表 → 用户详情（跳转关系）

原文：「详情页顶部显示返回按钮」
提取：用户详情 ← 用户列表（返回路径）
```

**页面关系类型：**

| 关系 | 表示方法 | 说明 |
|-----|---------|------|
| 父子关系 | A ⊃ B | 列表 → 详情 |
| 跳转关系 | A → B | 按钮/链接触发 |
| 返回关系 | A ← B | 返回上一页 |
| 弹窗关系 | A ⊕ B | 弹窗覆盖 |
| 标签关系 | A ↔ B | 同级切换 |

### 导航结构输出

**导航图格式：**

```
主导航
├── 首页 ─────────────── 首页仪表盘
├── 用户管理
│   ├── 用户列表 ──────── 用户详情（跳转）
│   │                 └── 用户表单（跳转）
│   └── 角色列表 ──────── 权限配置（跳转）
└── 系统设置
    └── 账户设置

全局元素
├── 顶部导航栏
├── 侧边菜单
└── 用户信息下拉
```

## 页面元素清单

### 元素提取模板

**页面元素清单格式：**

```markdown
## 页面：{页面名称}

### 基本信息
- 页面类型：列表页/详情页/表单页/仪表盘
- 所属模块：{模块名称}
- 访问权限：{权限要求}

### 页面结构
| 区块 | 类型 | 说明 |
|-----|------|------|
| 顶部 | 操作栏 | 标题 + 主操作按钮 |
| 搜索区 | 筛选组 | 搜索条件 + 按钮 |
| 内容区 | 列表/表单 | 主要内容展示 |
| 底部 | 分页/操作 | 分页或底部操作 |

### 组件清单
| 组件 | 类型 | 必填 | 默认值 | 说明 |
|-----|------|-----|-------|------|
| 用户名输入框 | 文本输入 | 是 | - | 用户名搜索 |
| 状态筛选 | 下拉选择 | 否 | 全部 | 状态筛选 |
| 搜索按钮 | 主按钮 | - | - | 触发搜索 |
| 数据表格 | 表格 | - | - | 展示用户列表 |
| 操作按钮组 | 按钮组 | - | - | 查看/编辑/删除 |

### 交互说明
- 搜索 → 刷新表格数据
- 点击行 → 跳转详情页
- 删除 → 弹窗确认 → 更新列表
```

### 元素优先级排序

**页面布局优先级（从上到下）：**

1. **必需元素** - 核心功能，始终显示
2. **常用元素** - 高频使用，显眼位置
3. **辅助元素** - 补充功能，适当位置
4. **次要元素** - 低频使用，折叠或底部

**组件排列原则：**
- 主操作在右侧/上方
- 次操作在左侧/下方
- 筛选在内容上方
- 分页在内容下方

## 状态与变体提取

### 页面状态识别

| 状态类型 | 触发条件 | UI表现 |
|---------|---------|--------|
| 初始状态 | 页面加载完成 | 默认数据展示 |
| 空状态 | 数据为空 | 空状态提示 |
| 加载状态 | 数据请求中 | 加载动画 |
| 错误状态 | 请求失败 | 错误提示 |
| 成功状态 | 操作成功 | 成功提示 |

### 组件变体识别

**从需求提取变体：**

```markdown
原文：「管理员可以看到所有用户，普通用户只能看到自己部门的用户」
提取：用户列表组件有两个变体
  - 变体A（管理员）：显示全部用户
  - 变体B（普通用户）：显示部门用户

原文：「禁用状态的用户显示灰色」
提取：用户行组件有状态变体
  - 启用状态：正常显示
  - 禁用状态：灰色显示
```

## 常见错误

- 层级过深（超过4级），用户难以理解
- 同一模块内页面命名不一致
- 忽略页面的多个状态变体
- 未识别公共组件，造成重复定义
- 导航结构过于复杂，超过7个主入口
- 混淆页面跳转和弹窗交互
