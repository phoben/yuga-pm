# 任务规划规则

本文档定义页面级任务规划流程、Git 分支策略和任务文档模板。

---

## 任务规划流程

### 流程概览

```
┌─────────────────────────────────────────────────────────────┐
│                    任务规划流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Step 1: 提取页面清单                                        │
│  ├── 从 DRD 读取所有页面                                    │
│  ├── 标记页面优先级                                         │
│  └── 识别页面依赖关系                                       │
│                                                             │
│  Step 2: 生成任务文档                                        │
│  ├── 每个页面生成一个任务文档                                │
│  └── 保存到 .yg-pm/projects/{project-id}/tasks/            │
│                                                             │
│  Step 3: 创建任务列表                                        │
│  ├── 使用 TaskCreate 创建任务                               │
│  └── 设置任务依赖关系                                       │
│                                                             │
│  Step 4: 分配开发策略                                        │
│  ├── 确定并行/串行策略                                      │
│  └── 分配 Coder Agent                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Step 1: 提取页面清单

从 DRD 文档中提取所有页面信息：

1. **读取页面列表**
   - 页面名称
   - 页面类型（列表页/详情页/表单页/仪表盘）
   - 页面优先级
   - 页面依赖

2. **标记页面优先级**

| 优先级 | 判断标准 | 示例 |
|-------|---------|------|
| High | 核心功能页面、入口页面 | 首页、登录页、主列表页 |
| Medium | 功能支持页面 | 详情页、设置页 |
| Low | 辅助功能页面 | 帮助页、关于页 |

3. **识别页面依赖关系**

```
依赖类型:
├── 数据依赖: 页面 B 需要页面 A 的数据
├── 组件依赖: 页面 B 使用页面 A 创建的共享组件
└── 导航依赖: 页面 B 从页面 A 导航进入

示例:
用户详情页 ──依赖──> 用户列表页 (导航依赖)
订单详情页 ──依赖──> 用户卡片组件 (组件依赖)
```

### Step 2: 生成任务文档

为每个页面生成独立的任务文档：

- 文档命名: `T{序号}-{page-name}.md`
- 保存位置: `.yg-pm/projects/{project-id}/tasks/`

### Step 3: 创建任务列表

使用 TaskCreate 工具创建任务并设置依赖：

```javascript
// 创建有依赖关系的任务
TaskCreate({
  id: "T001",
  title: "用户列表页面",
  dependencies: []  // 无依赖
})

TaskCreate({
  id: "T002",
  title: "用户详情页面",
  dependencies: ["T001"]  // 依赖 T001
})
```

### Step 4: 分配开发策略

根据页面数量和依赖关系选择策略：

| 页面数 | 策略 | 并行数 | 说明 |
|-------|------|-------|------|
| 1-3 页 | 全部并行 | 全部 | 无需等待 |
| 4-8 页 | 分批并行 | 4/批 | 按优先级分批 |
| 9+ 页 | 依赖优先 | 5/批 | 先开发被依赖页面 |

## Git 分支策略

### 分支命名规范

| 场景 | 分支命名模式 | 示例 |
|-----|-------------|------|
| 功能页面 | `feature/page-{name}` | `feature/page-user-list` |
| 共享组件 | `feature/component-{name}` | `feature/component-user-card` |
| Bug 修复 | `fix/{desc}` | `fix/login-validation` |
| 样式调整 | `style/{desc}` | `style/header-spacing` |
| 重构 | `refactor/{desc}` | `refactor/api-layer` |

### 分支命名规则

- 使用小写字母和连字符
- 名称简洁明了，能反映功能内容
- 使用英文，遵循 kebab-case 命名风格
- 避免使用特殊字符和数字开头

### 分支创建时机

**重要：** 分支由 Coder Agent 创建，而非 Main Agent。

```
正确流程:
Main Agent ──派发任务──> Coder Agent ──创建分支──> 开发

反模式:
Main Agent ──创建分支──> 派发任务 ──> Coder Agent
(问题: 如果 Agent 失败，会产生孤立分支)
```

### 分支工作流

```
main (主分支)
  │
  ├── feature/page-user-list
  │     └── 开发用户列表页面
  │           └── PR #1 ──审查──> 合并
  │
  ├── feature/page-user-detail
  │     └── 开发用户详情页面
  │           └── PR #2 ──审查──> 合并
  │
  └── feature/component-user-card
        └── 开发用户卡片组件
              └── PR #3 ──审查──> 合并
```

## 反模式

- 忽略页面依赖关系，导致开发顺序混乱
- 分支命名不一致，难以追踪
- 任务文档缺失关键信息
- Main Agent 提前创建分支
- 超过 5 个 Agent 并行运行
- 未设置任务依赖关系

## 示例：完整任务规划

### 场景：用户管理系统

**DRD 页面清单：**
1. 登录页 (High, 无依赖)
2. 首页仪表盘 (High, 依赖登录页)
3. 用户列表页 (High, 依赖登录页)
4. 用户详情页 (Medium, 依赖用户列表页)
5. 系统设置页 (Low, 依赖登录页)

**任务规划：**

```
Batch 1 (并行):
├── T001: 登录页 (feature/page-login)
├── T002: 首页仪表盘 (feature/page-dashboard)
└── T003: 用户列表页 (feature/page-user-list)

Batch 2 (T001-T003 完成后):
├── T004: 用户详情页 (feature/page-user-detail)
└── T005: 系统设置页 (feature/page-settings)
```

**任务文档示例 (T003)：**

```markdown
# Task: 用户列表页

## 元信息
- 任务 ID: T003
- 页面 ID: P003
- 类型: List
- 优先级: High
- 依赖: [T001]

### 布局要求
- 响应式: 支持 1440px/1024px/768px
- 布局: 筛选区 + 表格 + 分页

### 样式规范
- 主色: #1890ff
- 表格行高: 54px
- 边框颜色: #f0f0f0

## 组件清单

| 组件 | 类型 | 说明 |
|-----|------|------|
| UserFilter | 业务组件 | 搜索筛选栏 |
| UserTable | 业务组件 | 用户数据表格 |
| Pagination | 基础组件 | 分页组件 |

## 交付标准

- [ ] 页面渲染正确
```
