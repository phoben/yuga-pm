# 规则：文档分析

**优先级：** 关键 (CRITICAL)
**类别：** 文档分析

## 描述

文档分析是原型设计的基础。它确保从需求文档中准确提取页面结构、UI元素和交互逻辑，为原型生成提供可靠的输入。

## 需求文档解析

### 输入类型识别

| 文档类型 | 特征 | 解析重点 |
|---------|------|---------|
| DRD设计需求文档 | 包含页面结构、组件说明 | 直接映射到原型结构 |
| PRD产品需求文档 | 功能描述、业务逻辑 | 提取页面和交互流程 |
| TRD技术需求文档 | 数据结构、接口定义 | 提取数据展示形式 |
| 功能列表 | 功能点罗列 | 推断页面和模块 |

### 文档结构解析

**识别层级结构：**
1. **项目/产品层级** - 整体范围界定
2. **模块/功能层级** - 功能分组
3. **页面/界面层级** - 原型基本单位
4. **元素/组件层级** - UI构成单元

**解析步骤：**
1. 扫描文档标题结构，建立层级关系
2. 识别页面相关关键词：「页面」「界面」「屏」「视图」
3. 提取页面描述段落，标记核心内容
4. 识别跨页面引用和导航关系

### 元素识别规则

**UI元素关键词映射：**

| 元素类型 | 关键词 | 识别规则 |
|---------|-------|---------|
| 页面 | 页面、界面、屏、视图 | 作为顶层容器 |
| 表单 | 填写、输入、提交、表单 | 识别输入项和按钮 |
| 列表 | 列表、表格、数据、记录 | 识别列字段和操作 |
| 按钮 | 点击、按钮、操作、提交 | 识别触发动作 |
| 导航 | 菜单、导航、跳转、链接 | 识别跳转目标 |
| 卡片 | 卡片、区块、模块 | 识别内容边界 |
| 图表 | 图表、统计、数据可视化 | 识别数据类型 |

**属性提取：**
- 必填/选填标识 → 表单验证
- 默认值 → 初始状态
- 限制条件 → 输入约束
- 操作权限 → 显示/隐藏条件

## 交互逻辑提取

### 流程识别

**识别交互触发点：**
- 用户主动操作：点击、输入、滑动
- 系统响应：提示、跳转、刷新
- 条件触发：满足条件时执行

**交互类型分类：**

| 类型 | 描述 | 原型表现 |
|-----|------|---------|
| 页面跳转 | 从A页面到B页面 | 连接线标注 |
| 弹窗交互 | 模态/非模态弹出 | 弹窗状态 + 触发条件 |
| 局部更新 | 页面部分内容变化 | 状态变体标注 |
| 表单提交 | 数据提交处理 | 提交状态 + 结果反馈 |
| 筛选排序 | 列表数据变化 | 筛选条件 + 结果展示 |

### 条件逻辑识别

**从需求中提取条件：**
```
原文：「当用户未登录时，显示登录提示」
提取：条件(未登录) → 动作(显示登录提示)

原文：「提交成功后跳转到列表页」
提取：事件(提交成功) → 动作(跳转列表页)
```

**常见条件模式：**
- 权限条件：登录/未登录、角色权限
- 状态条件：数据为空、加载中、操作完成
- 数据条件：筛选结果、搜索结果
- 时间条件：倒计时、定时任务

## 识别示例

### 良好的需求描述

```markdown
## 用户列表页面

### 功能说明
展示系统中所有用户，支持搜索、筛选和导出。

### 页面元素
1. 搜索栏：输入框（用户名/手机号）+ 搜索按钮
2. 筛选区：状态下拉框、角色下拉框、日期范围选择
3. 数据表格：用户名、手机号、角色、状态、注册时间、操作
4. 操作列：查看详情、编辑、禁用/启用
5. 分页：每页10条，显示总数

### 交互说明
- 点击「搜索」按钮，根据条件刷新表格
- 点击「查看详情」跳转到用户详情页
- 点击「禁用」，弹出确认弹窗，确认后更新状态
```

**解析结果：**
- 页面：用户列表页
- 组件：搜索栏、筛选区、数据表格、分页
- 交互：搜索刷新、详情跳转、禁用确认

### 模糊的需求描述

```markdown
## 用户管理

用户可以查看和管理用户，支持搜索和操作。
```

**问题：**
- 未明确页面数量
- 未说明具体操作类型
- 未定义数据展示形式

**处理方式：**
1. 标记为「待确认」
2. 提供默认方案供选择
3. 记录假设条件

## 常见错误

- 跳过文档分析直接开始设计，导致遗漏功能
- 仅关注功能描述，忽略异常状态和边界条件
- 混淆页面层级，将模块当作页面处理
- 忽略文档中的条件描述，造成交互逻辑缺失
- 未识别数据依赖关系，导致页面间数据流断裂
