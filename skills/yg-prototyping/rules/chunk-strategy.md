# 分块策略规则

> 本规则定义了原型设计技能如何根据文档规模自动选择分块策略，以支持大规模需求文档的处理。

---

## 概述

分块策略是为了解决大规模需求文档（50-100页）处理时可能出现的超时问题。通过智能分块，将大文档拆分为多个可并行处理的片段，由多个子Agent协同完成。

---

## 分块原则

### 语义完整性

- 优先保持页面/模块的语义完整性
- 同一页面的内容不应跨块分割
- 相关的页面尽量放在同一块

### 负载均衡

- 各块的字符数尽量均匀
- 避免出现超大块或超小块
- 每块约 15000 字符（约 6 页）

### 可扩展性

- 支持动态调整分块数量
- 根据系统负载自动优化
- 预留扩展接口

---

## 分块策略矩阵

### 按文档规模选择策略

| 文档规模 | 分块模式 | 每块大小 | Agent数量 | 说明 |
|---------|---------|---------|----------|------|
| ≤20页 | 直接处理 | 全部 | 1 | 无需分块，单Agent处理 |
| 21-50页 | 按模块 | 按模块边界 | 2-5 | 按模块/章节分块 |
| 51-100页 | 按页面 | 6页/块 | 8-17 | 按页面分块并行处理 |
| >100页 | 按页面+并行 | 6页/块 | 动态 | 动态调整，分批处理 |

### 文档规模估算

```
文档规模 = 总页数 ≈ 总字符数 / 2500
         ≈ 总行数 / 50
```

## 分块方法

### 方法一：页面标记识别（优先）

**适用场景：** 文档中有明确的页面/模块标记

**识别关键词：**
- 页面标记：「页面」「界面」「屏」「视图」「Page」
- 模块标记：「模块」「功能」「Module」「Feature」
- 章节标记：「##」「###」等标题层级

**分块步骤：**

```
1. 扫描文档，识别所有页面标记
2. 建立页面索引：{页面名: 起始位置, 结束位置}
3. 按页面边界进行分组
4. 确保每组字符数在合理范围内
```

**示例：**

```markdown
## 用户管理模块          ← 模块边界
### 用户列表页面          ← 页面边界
[内容...]
### 用户详情页面          ← 页面边界
[内容...]
## 订单管理模块          ← 模块边界
### 订单列表页面          ← 页面边界
[内容...]
```

### 方法二：字符数兜底

**适用场景：** 文档无明确标记，或标记不规范

**分块算法：**

```python
def split_by_chars(content, chunk_size=15000):
    """
    按字符数分块，优先在标题处断开
    """
    chunks = []
    current_chunk = ""

    for section in content.split('\n##'):
        if len(current_chunk) + len(section) > chunk_size:
            if current_chunk:
                chunks.append(current_chunk)
            current_chunk = section
        else:
            current_chunk += '\n##' + section

    if current_chunk:
        chunks.append(current_chunk)

    return chunks
```

**断点优先级：**
1. 一级标题 `# `
2. 二级标题 `## `
3. 三级标题 `### `
4. 空行分隔
5. 强制断点

## 分块元数据

每个分块需要携带以下元数据：

```json
{
  "chunk_id": "chunk_001",
  "chunk_index": 0,
  "total_chunks": 5,
  "char_count": 14800,
  "page_range": [1, 6],
  "pages": ["用户列表", "用户详情"],
  "style_ref": "./styles.css",
  "dependencies": ["common_components"]
}
```

## 分块执行流程

```
┌─────────────────────────────────────┐
│           读取需求文档               │
└─────────────────┬───────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│         计算文档规模                 │
│    总字符数 / 总行数 / 标题数量      │
└─────────────────┬───────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│         选择分块策略                 │
│  ≤20页 → 直接处理                    │
│  21-50页 → 按模块                    │
│  51-100页 → 按页面                   │
└─────────────────┬───────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│         执行分块                     │
│  优先：页面标记识别                  │
│  兜底：字符数分割                    │
└─────────────────┬───────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│         生成分块元数据               │
│  chunk_id, pages, dependencies...   │
└─────────────────┬───────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│         分发到子Agent                │
└─────────────────────────────────────┘
```

## 分块边界处理

### 跨块引用处理

当页面A引用了页面B，但它们被分到不同块时：

```json
{
  "cross_references": [
    {
      "from_chunk": "chunk_001",
      "from_page": "用户列表",
      "to_page": "用户详情",
      "to_chunk": "chunk_002",
      "reference_type": "navigation"
    }
  ]
}
```

主控Agent负责在合并阶段处理跨块引用。

### 公共组件处理

识别跨页面使用的公共组件：

```json
{
  "shared_components": [
    {
      "name": "用户选择器",
      "used_in_pages": ["订单创建", "任务分配"],
      "used_in_chunks": ["chunk_001", "chunk_003"]
    }
  ]
}
```

公共组件由主控Agent统一生成，子Agent引用。

## 配置选项

分块策略支持以下配置：

| 配置项 | 默认值 | 说明 |
|-------|-------|------|
| `chunk_size` | 15000 | 每块目标字符数 |
| `max_chunks` | 20 | 最大分块数量 |
| `min_chunk_size` | 5000 | 最小块大小 |
| `prefer_semantic` | true | 优先语义边界 |
| `parallel_threshold` | 21 | 启用并行的页数阈值 |

---

## 监控指标

| 指标 | 说明 | 目标值 |
|-----|------|-------|
| 分块均匀度 | 各块字符数标准差 | < 20% |
| 语义完整率 | 未跨页分割的比例 | > 95% |
| 分块耗时 | 分块操作耗时 | < 5s |
| 并行效率 | 并行加速比 | > 0.7 |

---

## 反模式

- ❌ 将单页面内容分割到多个块
- ❌ 分块数量过多（> 20 块）
- ❌ 分块大小差异过大（最大/最小 > 3倍）
- ❌ 忽略跨块引用关系
- ❌ 在表格/代码块中间断开
