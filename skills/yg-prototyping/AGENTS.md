# 原型设计 — 完整规则汇编

本文档整合了原型设计技能的所有规则、指南和最佳实践，按优先级组织。

---

## 前置：技术栈确认

### 技术栈决策

| 项目类型 | 推荐技术栈 | 说明 |
|---------|-----------|------|
| Web后台管理系统 | **React + Element UI** | 组件丰富，默认选择 |
| 小程序 | Taro + Ant Design Mini | 跨端支持 |
| 移动端H5/APP | React + Ant Design Mobile | 移动端优化 |
| 简单原型 | React + shadcn/ui | 轻量现代 |
| 单页面原型 | HTML + CSS 原生 | 无依赖 |

### 环境评估检查

- Node.js 版本 >= 18
- 包管理器（pnpm/npm）
- 相关 Skills 是否安装
- MCP 工具是否配置

**处理策略：** 环境不满足时引导安装、提供降级方案或更换技术栈

## 优先级 1：文档分析（关键）

> 详细内容：[rules/document-analysis.md](rules/document-analysis.md)

### 需求文档解析
- 首先识别文档类型（DRD/PRD/TRD/功能列表）
- 扫描文档标题结构，建立层级关系
- 识别页面相关关键词：「页面」「界面」「屏」「视图」
- 提取跨页面引用和导航关系

### UI元素识别
- 建立关键词到组件类型的映射表
- 从描述中提取属性：必填/选填、默认值、限制条件
- 识别交互触发点：用户操作、系统响应、条件触发

### 交互逻辑提取
- 流程识别：页面跳转、弹窗交互、局部更新、表单提交
- 条件逻辑：权限条件、状态条件、数据条件、时间条件

**反模式：** 跳过文档分析直接设计 · 仅关注功能忽略异常状态 · 混淆页面层级

## 优先级 2：结构提取（关键）

> 详细内容：[rules/structure-extraction.md](rules/structure-extraction.md)

### 页面层级结构
- 三级结构模型：应用 → 模块 → 页面 → 区块 → 组件
- 层级不超过4级，避免用户难以理解
- 页面命名规范：列表页、详情页、表单页、设置页、仪表盘

### 导航关系梳理
- 主导航固定入口，覆盖核心功能
- 导航类型：顶部导航、侧边导航、底部导航、面包屑、标签导航
- 导航入口不超过7个

### 页面元素清单
- 基本信息：页面类型、所属模块、访问权限
- 页面结构：区块划分和类型说明
- 组件清单：组件类型、必填、默认值、说明
- 交互说明：触发事件和响应行为

**反模式：** 层级过深 · 命名不一致 · 忽略状态变体 · 未识别公共组件

---

## 优先级 3：布局设计（高）

> 详细内容：[rules/layout-design.md](rules/layout-design.md)

### 布局模式选择
- 顶部导航布局：网站、营销页
- 侧边栏布局：后台管理系统（功能7-15个）
- 顶部+侧边布局：复杂管理系统（功能超过15个）
- 单页布局：落地页、展示页

### 栅格系统
- 移动端：4列，间距16px
- 平板：8列，间距24px
- 桌面端：12列，间距24px
- 基础间距单位：8px

### 页面区域划分
- 页面标题区（padding: 16-24px）
- 筛选/搜索区（可选）
- 主要内容区
- 分页/底部操作（可选）

**反模式：** 布局拥挤缺少留白 · 导航层级过多 · 忽略响应式 · 对齐不一致

## 优先级 4：组件放置（高）

> 详细内容：[rules/component-placement.md](rules/component-placement.md)

### 组件选型
- 输入文本 → 文本输入框/文本域
- 选择选项 → 下拉选择器/单选按钮组
- 多选 → 复选框组/多选下拉
- 触发操作 → 按钮/链接
- 数据列表 → 表格/卡片列表

### 位置原则
- F型阅读模式：左上角最重要信息
- 主操作：页面右上角或表单底部
- 次操作：主操作左侧或下方
- 每区块最多1个主操作按钮

### 组件状态设计
| 状态 | 触发条件 | 视觉表现 |
|-----|---------|---------|
| 默认 | 初始状态 | 标准样式 |
| 悬停 | 鼠标悬停 | 背景变深/边框高亮 |
| 禁用 | 不可操作 | 灰色/透明度降低 |
| 加载 | 处理中 | 加载动画 |
| 错误 | 验证失败 | 红色边框/错误提示 |

**反模式：** 组件类型选择不当 · 按钮过多主次不分 · 表单排列混乱 · 忽略状态变体

---

## 优先级 5：交互标注（中等）

> 详细内容：[rules/interaction-annotation.md](rules/interaction-annotation.md)

### 标注类型
- **页面级标注：** 页面名称、入口、权限、状态说明
- **组件级标注：** 组件名称、数据来源、交互行为、状态变化
- **流程级标注：** 流程名称、起止点、决策分支、异常处理

### 连接线规范
| 类型 | 线条样式 | 含义 |
|-----|---------|------|
| 实线箭头 | ──────→ | 页面跳转 |
| 虚线箭头 | - - - - → | 弹窗打开 |
| 双向箭头 | ←──────→ | 标签切换 |

### 标注符号系统
- 🔵 默认状态 · 🟢 成功状态 · 🟡 警告状态 · 🔴 错误状态
- ⚪ 禁用状态 · ⏳ 加载状态
- 👆 点击 · ⌨️ 输入 · 🖱️ 悬停 · 📱 触摸

**反模式：** 标注过于简单 · 标注与原型分离 · 忽略异常状态 · 使用模糊术语

## 协作建议

### 与产品经理协作
- 确认需求理解的准确性
- 标注待确认的模糊点
- 提供多种方案供选择

### 与设计师协作
- 原型作为设计参考起点
- 保留交互逻辑说明
- 注明设计自由度范围

### 与开发协作
- 标注关键交互逻辑
- 说明数据来源和格式
- 注明边界条件和异常处理
