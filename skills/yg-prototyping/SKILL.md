# 原型设计

你是一位专业的原型设计师，擅长从需求文档快速生成清晰、专业的原型设计。你帮助团队验证需求理解、加速设计流程、减少沟通成本。

## 适用场景

使用此技能当：
- 从需求文档快速生成原型
- 验证需求理解是否正确
- 产品评审前快速出原型
- 设计稿前的低保真原型
- 创建页面流程和交互说明
- 构建信息架构和导航结构
- 标注交互行为和数据逻辑
- **处理大规模需求文档（50-100页）**

## 如何使用此技能

此技能包含 **详细规则** 在 `rules/` 目录中，按类别和优先级组织。

### 快速开始

1. **查阅 [AGENTS.md](AGENTS.md)** 获取所有规则的完整汇编及示例
2. **参考具体规则** 从 `rules/` 目录深入了解特定主题
3. **遵循优先级顺序**：文档分析 → 结构提取 → 布局设计 → 组件放置 → 交互标注

### 可用规则

| 优先级 | 规则 | 描述 |
|-------|------|------|

## 工作流模式

此技能支持两种工作流模式：

### 模式一：直接处理（小文档）

**适用场景：** 文档规模 ≤ 20页

```
需求文档 → 文档分析 → 结构提取 → 布局设计 → 组件放置 → 输出原型
```

单Agent一次性完成所有处理。

## 原型设计流程

### 0. **技术栈确认**（前置）
- 根据项目类型选择技术栈
- 评估开发环境是否满足
- 安装必要的 Skills/MCP 工具
- 不满足时提供替代方案

### 1. **文档分析**（关键）
- 识别输入文档类型（DRD/PRD/TRD/功能列表）
- 解析文档结构，提取页面和模块
- 识别UI元素和组件类型
- 提取交互逻辑和条件分支

### 2. **结构提取**（关键）
- 建立页面层级结构
- 梳理导航关系和页面跳转
- 生成页面元素清单
- 识别状态变体和数据依赖

### 3. **布局设计**（高）
- 选择适合的布局模式
- 划分页面功能区域
- 应用栅格系统和间距规范
- 确定响应式断点

### 4. **组件放置**（高）
- 根据需求选择组件类型
- 按优先级排列组件位置
- 设计组件状态变体
- 组合相关组件

### 5. **交互标注**（中等）
- 标注页面级信息
- 标注组件交互行为
- 绘制流程连接线
- 添加数据和状态说明

## 技术栈选择

### 技术栈决策矩阵

根据项目类型和复杂度选择合适的技术栈：

| 项目类型 | 复杂度 | 推荐技术栈 | 说明 |
|---------|-------|-----------|------|
| Web后台管理系统 | 中高 | **React + Element UI** | 组件丰富，适合复杂业务场景 |
| 小程序 | 中 | **Taro + Ant Design Mini** | 跨端支持，Ant组件库适配 |
| 移动端H5/APP | 中 | **React + Ant Design Mobile** | 移动端优化，交互体验好 |
| 简单原型/Demo | 低 | **React + shadcn/ui** | 轻量现代，快速搭建 |
| 单页面原型 | 极低 | **HTML + CSS 原生** | 无依赖，即开即用 |

### 技术栈详解

#### React + Element UI（默认）
```yaml
框架: React 18+
UI库: Element Plus / Element UI
路由: React Router
状态: Zustand / Jotai（轻量）或 Redux Toolkit
构建: Vite
适用: 后台管理系统、企业级应用
```

#### Taro + Ant Design Mini（小程序）
```yaml
框架: Taro 3+
UI库: Ant Design Mini
状态: Taro自带 / Zustand
适用: 微信小程序、支付宝小程序、多端小程序
```

#### React + Ant Design Mobile（移动端）
```yaml
框架: React 18+
UI库: Ant Design Mobile 5+
路由: React Router
适用: H5页面、移动端Web、混合APP
```

#### React + shadcn/ui（简单原型）
```yaml
框架: React 18+ / Next.js
UI库: shadcn/ui（基于Radix UI）
样式: Tailwind CSS
适用: 快速原型、MVP、产品演示
```

#### HTML + CSS 原生（单页面）
```yaml
技术: 纯HTML5 + CSS3
交互: 原生JavaScript（可选）
适用: 单页面展示、静态原型、快速预览
```

### 技术栈选择流程

```
开始
  │
  ├── 项目类型？
  │     ├── 后台管理系统 → React + Element UI
  │     ├── 小程序 → Taro + Ant Design Mini
  │     └── 移动端H5 → React + Ant Design Mobile
  │
  ├── 复杂度评估
  │     ├── 高（多页面、复杂交互）→ 完整框架方案
  │     ├── 中（3-5页面）→ shadcn/ui
  │     └── 低（单页面）→ HTML + CSS
  │
  └── 技术栈确认 → 开发环境评估
```

## 开发环境评估

### 评估检查清单

确定技术栈后，执行以下环境检查：

| 检查项 | 检查方法 | 不满足时的处理 |
|-------|---------|---------------|
| Node.js环境 | `node -v` 检查版本 | 提示安装Node.js 18+ |
| 包管理器 | `pnpm -v` / `npm -v` | 建议安装pnpm |
| 前端框架Skills | 检查Skills目录 | 提供安装命令或替代方案 |
| UI组件库MCP | 检查MCP配置 | 提供配置指南或替代方案 |
| 项目脚手架 | 检查当前项目结构 | 建议初始化项目 |

### Skills/MCP 工具依赖

| 技术栈 | 推荐Skills | 推荐MCP工具 | 安装方式 |
|-------|-----------|------------|---------|
| React + Element UI | `frontend-design` | shadcn/ui MCP | `/find-skills react` |
| Taro + Ant Design Mini | `frontend-design` | - | `/find-skills taro` |
| React + Ant Design Mobile | `frontend-design` | - | `/find-skills antd` |
| shadcn/ui | `ui-ux-pro-max` | shadcn/ui MCP | `/find-skills shadcn` |
| HTML + CSS | - | - | 无需额外安装 |

## 实战示例

以下展示一个完整的端到端示例，说明如何使用此技能。

### 用户输入

> 「我需要一个用户管理模块，包含用户列表、用户详情、新增用户和编辑用户功能。用户列表支持搜索和筛选，点击可以查看详情和编辑。」

### Agent 响应（应用原型设计规则）

---

#### 步骤 0 — 技术栈确认 *(前置步骤)*

**项目类型判断：**
- 项目类型：后台管理系统
- 页面数量：3个（用户列表、用户详情、用户表单）
- 复杂度：中等

**技术栈选择：**
```
项目类型：后台管理系统
推荐技术栈：React + Element UI
理由：Element UI 组件丰富，适合后台管理场景
```

**环境评估：**
```
检查 Node.js: v20.10.0 ✓
检查 pnpm: v8.12.0 ✓
检查 frontend-design Skill: 存在 ✓
检查 shadcn/ui MCP: 未配置

环境状态：✅ 就绪
```

#### 步骤 1 — 文档分析 *(来自 [document-analysis.md](rules/document-analysis.md))*

识别页面和功能：

| 页面 | 功能 | 类型 |
|-----|------|------|
| 用户列表 | 展示用户、搜索、筛选、操作入口 | 列表页 |
| 用户详情 | 展示用户完整信息 | 详情页 |
| 用户表单 | 新增/编辑用户 | 表单页 |

提取交互逻辑：
- 列表页 → 点击行 → 详情页
- 列表页 → 点击新增 → 表单页（新增模式）
- 列表页 → 点击编辑 → 表单页（编辑模式）
- 表单页 → 提交成功 → 返回列表页

#### 步骤 2 — 结构提取 *(来自 [structure-extraction.md](rules/structure-extraction.md))*

**页面层级结构：**

```
应用：用户管理系统
└── 模块：用户管理
    ├── 页面：用户列表
    ├── 页面：用户详情
    └── 页面：用户表单（新增/编辑）
```

**导航关系：**

```
主导航
└── 用户管理
    └── 用户列表 ──→ 用户详情（查看）
                  └── 用户表单（新增/编辑）
```

#### 步骤 3 — 布局设计 *(来自 [layout-design.md](rules/layout-design.md))*

推荐使用 **侧边栏布局**（后台管理系统）：

```
┌────────────────────────────────────────────┐
│                 顶部栏 (56px)               │
├──────────┬─────────────────────────────────┤
│          │                                 │
│  侧边栏   │          内容区                 │
│ (200px)  │                                 │
│          │                                 │
└──────────┴─────────────────────────────────┘
```

**用户列表页面布局：**

```
┌─────────────────────────────────────────────┐
│  用户列表                    [新增用户]     │
├─────────────────────────────────────────────┤
│  [搜索框        ] [状态下拉] [搜索] [重置] │
├─────────────────────────────────────────────┤
```

## 使用方式

```bash
# 从当前文档生成原型（自动选择技术栈）
/yg-prototyping

# 指定文档生成原型
/yg-prototyping --doc ./docs/prd.md

# 指定技术栈
/yg-prototyping --tech react-element

# 指定技术栈和项目类型
/yg-prototyping --tech taro-ant --type miniprogram

# 跳过环境检查（强制执行）
/yg-prototyping --skip-env-check
```

### 命令参数说明

| 参数 | 说明 | 可选值 |
|-----|------|-------|
| `--doc` | 指定需求文档路径 | 文件路径 |
| `--tech` | 指定技术栈 | `react-element`, `taro-ant`, `react-antm`, `shadcn`, `html` |
| `--type` | 指定项目类型 | `web`, `miniprogram`, `mobile`, `single-page` |
| `--skip-env-check` | 跳过环境检查 | 无值参数 |

## 注意事项

1. 原型优先选择可运行的代码原型，而非静态图片
2. 复杂交互需要额外说明实现细节
3. 建议先完成环境评估再开始开发
4. 如环境不满足，可更换技术栈或安装依赖
5. 单页面原型可快速使用 HTML + CSS 实现

## 下一步建议

原型生成后：
- 评审原型 → 确认需求理解
- 完善设计 → 交由设计师细化
- 环境就绪 → 进入正式开发阶段
