# 上下文注入规则

## 注入优先级

```
相关原始需求（核心） → 术语表（强制） → 前序章节（智能）
```

## 注入内容清单

| 注入项 | 必要性 | 说明 |
|--------|--------|------|
| 大纲 | 必须 | 完整大纲，帮助SubAgent了解全局结构 |
| 术语表 | 必须 | 强制使用统一定义的术语 |
| 章节模板 | 必须 | 该章节的格式要求 |
| 相关原始需求 | 必须 | 根据需求映射表注入相关需求片段 |
| 前序章节 | 智能注入 | 根据耦合度决定注入量 |

## 智能注入策略

### 耦合度计算

```python
def calculate_coupling(chapter, previous_chapters):
    """
    计算章节与前序章节的耦合度
    返回值: 0.0 - 1.0
    """
    coupling_score = 0.0

    # 检查交叉引用
    if has_cross_reference(chapter, previous_chapters):
        coupling_score += 0.3

    # 检查术语依赖
    if shares_terminology(chapter, previous_chapters):
        coupling_score += 0.2

    # 检查数据流依赖
    if has_data_dependency(chapter, previous_chapters):
        coupling_score += 0.3

    # 检查流程依赖
    if has_process_dependency(chapter, previous_chapters):
        coupling_score += 0.2

    return min(coupling_score, 1.0)
```

### 注入量决策

| 耦合度 | 注入内容 | 适用场景 |
|--------|----------|----------|
| > 0.7（高耦合） | 前序章节全文 | 数据模型依赖、流程延续 |
| 0.3 - 0.7（中耦合） | 前序章节摘要 | 术语关联、功能关联 |
| < 0.3（低耦合） | 仅章节标题 | 独立模块、附录类章节 |

## 上下文注入模板

```
## 章节信息
- 章节名称：[章节名称]
- 章节编号：[章节编号]
- 模板要求：[模板要求]

## 大纲（必须遵循）
[完整大纲]

## 术语表（强制使用）
[术语表]

## 本章关联的原始需求（核心输入）
[根据需求映射表注入相关原始需求片段]

## 前序章节上下文
[根据耦合度智能注入]

## 输出要求
- 完整覆盖上述"本章关联的原始需求"中的所有需求点
- 使用大纲中定义的术语，不得创造新术语
- 遵循模板格式要求
- 如有交叉引用，使用正确的章节编号
```

## 高耦合章节识别

以下情况视为高耦合，需要注入前序章节全文：

| 场景 | 说明 |
|------|------|
| 数据建模章节 | 需要了解前序功能需求中的数据特征 |
| 接口设计章节 | 需要了解前序数据模型的字段定义 |
| 流程描述章节 | 需要延续前序章节的业务流程 |
| 安全设计章节 | 需要了解前述功能的敏感操作 |

## 中耦合章节识别

以下情况视为中耦合，注入前序章节摘要：

| 场景 | 说明 |
|------|------|
| 功能模块章节 | 可能引用前序模块的功能定义 |
| 权限设计章节 | 需要了解角色定义但不需细节 |
| 报表设计章节 | 需要知道数据来源但不需要数据模型 |

## 低耦合章节识别

以下情况视为低耦合，仅注入章节标题：

| 场景 | 说明 |
|------|------|
| 概述章节 | 独立性高，无需前序上下文 |
| 附录章节 | 补充性质，独立完整 |
| 参考文献章节 | 完全独立 |
