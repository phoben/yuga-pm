# 需求映射规则

## 映射表结构

```markdown
## 需求映射表

| 章节 | 关联需求ID | 需求摘要 |
|------|------------|----------|
| 2.1 功能模块A | REQ-001, REQ-002 | 用户登录、权限验证 |
| 2.2 功能模块B | REQ-003, REQ-004 | 数据导入、数据导出 |
| 3.1 数据实体 | REQ-005 | 用户数据存储 |
```

## 映射表的作用

| 作用 | 说明 |
|------|------|
| **SubAgent上下文注入** | 根据映射表只注入相关需求，避免上下文撑满 |
| **需求覆盖验证** | 质量检查时核对每个需求是否被文档覆盖 |
| **需求追溯** | 确保每个需求都有对应的文档章节 |

## 建立映射的原则

### 原则1：一需求一章节

每个需求应映射到唯一的主要章节，避免重复覆盖。

```
✅ 正确：REQ-001 → 2.1 用户登录
❌ 错误：REQ-001 → 2.1 用户登录, 3.2 安全设计
```

### 原则2：完整性覆盖

所有原始需求都必须映射到某个章节。

```
检查方法：
原始需求总数 = 映射表中需求ID总数
```

### 原则3：合理归类

相关需求应归类到同一章节。

```
✅ 正确：
  REQ-001 用户登录 → 2.1 用户认证
  REQ-002 密码重置 → 2.1 用户认证
  REQ-003 权限管理 → 2.2 权限控制

❌ 错误：
  REQ-001 用户登录 → 2.1 概述
  REQ-002 密码重置 → 3.1 数据设计
```

## 映射表示例

| 章节 | 关联需求ID | 需求摘要 |
|------|------------|----------|

| 3. 数据需求 | REQ-009, REQ-010 | 数据存储、数据备份 |

|------|------------|----------|

## 需求注入格式

根据映射表，为SubAgent注入需求内容：

```
## 本章关联的原始需求

### REQ-001: 用户登录功能
用户应能够通过用户名和密码登录系统...

### REQ-002: 权限验证
系统应在用户执行敏感操作前验证其权限...
```

## 未映射需求处理

如果存在未映射的需求，在大纲确认环节提示：

```
⚠️ 检测到未映射的需求：
- REQ-XXX: [需求描述]

建议添加对应章节或合并到现有章节。
```
