## 活字格框架特征（自动排除项）

以下功能模块由活字格开发框架自带，在PRD设计环节**无需考虑**这些模块的功能、数据表、页面等实现，只需在文档中声明"由活字格框架实现"即可：

| 模块 | 说明 | PRD中的处理方式 |
|------|------|-----------------|
| 用户管理 | 用户增删改查、用户信息维护 | 声明"用户管理由活字格框架实现" |
| 角色管理 | 角色定义、角色权限配置 | 声明"角色管理由活字格框架实现" |
| 组织管理 | 组织架构、组织级别管理 | 声明"组织管理由活字格框架实现" |
| 用户鉴权/登录 | 登录、注销、会话管理 | 声明"用户鉴权由活字格框架实现" |
| 数据表行/列权限 | RBAC权限控制 | 声明"数据权限由活字格权限策略实现" |
| 审批工作流 | 可视化流程设计 | 声明"审批流程由活字格工作流引擎实现"（需说明流程节点） |
| 报表设计 | 可视化报表设计 | 声明"报表由活字格报表模块实现"（需说明报表数据源） |
| 审计日志 | 系统日志追踪 | 声明"审计日志由活字格管理后台实现" |
| 审计字段 | 活字格规范审计字段 | 核心业务表需遵循活字格标准审计字段命名规范 |

### 活字格标准审计字段

核心业务表需要手动创建以下审计字段，活字格会自动维护其值：

| 字段名 | 数据类型 | 用途说明 |
|--------|----------|----------|
| FGC_Creator | nvarchar | 创建人 |
| FGC_CreateDate | datetime | 创建时间 |
| FGC_LastModifier | nvarchar | 最后修改人 |
| FGC_LastModifyDate | datetime | 最后修改时间 |
| FGC_Rowversion | timestamp | 行版本号（乐观锁） |
