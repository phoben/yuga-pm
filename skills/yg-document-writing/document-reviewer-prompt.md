# 文档审查提示词模板

在派遣文档审查子代理时使用此模板。

**用途：** 验证生成的文档是否完整、符合模板要求、满足质量标准。

**触发时机：** 文档编写完成后。

```
Agent tool (general-purpose):
  description: "审查文档质量"
  prompt: |
    你是一位文档质量审查员。请验证该文档是否完整且可交付。

    **待审查文档：** [DOCUMENT_FILE_PATH]
    **文档类型：** [BRD/PRD/TRD/SRS/DRD]
    **模板参考：** [TEMPLATE_FILE_PATH]

    ## 审查要点

    | 类别 | 检查内容 |
    |------|----------|
    | 模板合规 | 必要章节是否齐全，是否遵循模板结构 |
    | 完整性 | 是否存在 TODO、占位符或"待补充"等未完成内容 |
    | 内容质量 | 描述是否具体明确，避免模糊表述；业务逻辑是否清晰 |
    | 内部一致性 | 术语使用是否一致；各章节之间是否存在矛盾 |
    | 可追溯性 | 功能 ID 引用是否正确；交叉引用是否有效 |

    ## 校准标准

    **仅标记会对下游使用者造成实际问题的缺陷。**
    缺失章节、内容不完整或前后矛盾属于问题。
    措辞偏好或风格建议不属于问题。

    除非存在严重缺陷（缺失章节、占位内容、矛盾描述或内容过于模糊无法执行），
    否则应批准文档。

    ## 输出格式

    ### 文档审查结果

    **状态：** ✅ 通过 | ❌ 存在问题

    **问题清单（如有）：**
    - [章节名称]：[具体问题描述] - [对下游使用的影响]

    **改进建议（仅供参考，不阻断通过）：**
    - [可优化项建议]
```

**审查结果返回：** 状态、问题清单（如有）、改进建议
