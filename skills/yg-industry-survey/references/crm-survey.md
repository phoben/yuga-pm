# CRM 行业知识库

## 一、行业概述

### 1.1 定义与范围

CRM（Customer Relationship Management，客户关系管理）是企业用于管理与客户交互的系统，覆盖销售、营销、客户服务三大核心领域。

**覆盖范围**：
- 销售自动化（SFA）
- 营销自动化
- 客户服务与支持
- 客户数据分析

### 1.2 核心特征

| 特征 | 说明 |
|------|------|
| 360° 客户视图 | 整合客户所有交互信息 |
| 流程自动化 | 销售流程、营销活动、服务流程自动化 |
| 数据驱动 | 基于数据分析优化客户互动 |
| 多渠道整合 | 整合邮件、电话、社交媒体等渠道 |

### 1.3 市场格局

- **主要玩家**：Salesforce, HubSpot, Microsoft Dynamics 365, Zoho CRM, Pipedrive
- **市场趋势**：AI 增强销售、全渠道客户体验、收入运营（RevOps）

## 二、核心模块

### 2.1 销售模块（Sales）

**功能说明**：
- **线索管理**：线索捕获、评分、分配、培育、转化
- **商机管理**：商机跟踪、阶段管理、赢单/输单分析
- **客户管理**：账户层级、联系人角色、交互历史
- **销售管道**：可视化管道、阶段转化率、预测分析
- **报价管理**：报价生成、定价规则、电子签名

**核心流程**：
```
线索捕获 → 线索评分 → 线索分配 → 线索培育 → MQL/SQL → 商机创建 → 阶段推进 → 成交/流失
```

**销售阶段模型**：
```
1. Prospecting (10%) → 2. Qualification (20%) → 3. Needs Analysis (40%)
→ 4. Proposal (60%) → 5. Negotiation (80%) → 6. Closed Won/Lost (100%/0%)
```

**关键指标**：
| 指标名称 | 计算方式 | 说明 |
|----------|----------|------|
| 线索转化率 | SQL数 / 总线索数 × 100% | 衡量线索质量 |
| 赢单率 | 赢单数 / 总商机数 × 100% | 衡量销售效率 |
| 平均成交金额 | 总收入 / 成交数 | 衡量交易规模 |
| 销售周期 | 商机创建到成交的平均天数 | 衡量销售效率 |
| 管道覆盖倍数 | 管道金额 / 目标金额 | 衡量管道健康度 |

### 2.2 营销模块（Marketing）

**功能说明**：
- **活动管理**：多渠道活动策划、执行、ROI 追踪
- **营销自动化**：邮件工作流、触发式营销、落地页
- **线索培育**：培育流、内容映射、互动追踪
- **细分管理**：人口统计、行为、企业属性细分

**核心流程**：
```
活动策划 → 受众细分 → 内容创建 → 多渠道执行 → 效果追踪 → ROI 分析
```

**关键指标**：
| 指标名称 | 计算方式 | 说明 |
|----------|----------|------|
| 邮件打开率 | 打开数 / 发送数 × 100% | 衡量邮件吸引力 |
| 点击率 | 点击数 / 发送数 × 100% | 衡量内容相关性 |
| 转化率 | 转化数 / 访问数 × 100% | 衡量落地页效果 |
| 单线索成本 | 营销投入 / 线索数 | 衡量获客效率 |
| 营销 ROI | (收入-成本) / 成本 × 100% | 衡量营销效益 |

### 2.3 客服模块（Service）

**功能说明**：
- **工单管理**：工单创建、分配、优先级、升级、解决
- **知识库**：文章创建、分类、搜索、评分
- **SLA 管理**：响应/解决时效、预警、报告
- **全渠道支持**：邮件、电话、在线聊天、社交媒体、自助门户

**核心流程**：
```
工单提交 → 路由分配 → 处理跟进 → 解决确认 → 满意度调查 → 关闭
```

**关键指标**：
| 指标名称 | 计算方式 | 说明 |
|----------|----------|------|
| 首次响应时间 | 首次响应的平均时长 | 衡量服务响应速度 |
| 平均解决时间 | 工单创建到关闭的平均时长 | 衡量服务效率 |
| 客户满意度 | 满意度评分平均值 | 衡量服务质量 |
| 首次解决率 | 首次解决数 / 总工单数 | 衡量问题解决能力 |
| SLA 达成率 | 达成SLA数 / 总工单数 × 100% | 衡量服务水平 |

## 三、数据模型

### 3.1 核心实体

| 实体名称 | 说明 | 核心字段 |
|----------|------|----------|
| Lead | 线索 | Source, Status, Score, Company, Contact Info, Owner |
| Account | 客户账户 | Name, Industry, Size, Type, Parent Account, Owner |
| Contact | 联系人 | Name, Title, Email, Phone, Account, Role, Owner |
| Opportunity | 商机 | Name, Account, Amount, Close Date, Stage, Probability, Owner |
| Activity | 活动 | Type (Task/Event/Email/Note), Date, Related To, Owner |
| Case | 工单 | Subject, Priority, Status, Origin, Contact, Owner |

### 3.2 实体关系

```
Lead --转化--> Opportunity
Account --1:N--> Contact
Account --1:N--> Opportunity
Account --1:N--> Case
Contact --1:N--> Opportunity
Contact --1:N--> Case
Opportunity --1:N--> Activity
```

## 四、业务流程

### 4.1 端到端流程

**线索到现金（Lead-to-Cash）**：
```
市场活动 → 线索捕获 → 线索培育 → 商机创建 → 报价提案 → 成交 → 回款
```

**流程说明**：
| 阶段 | 涉及角色 | 关键动作 | 输出物 |
|------|----------|----------|--------|
| 市场活动 | 营销 | 活动策划执行 | 潜在客户名单 |
| 线索捕获 | 营销/销售 | 表单提交、导入 | 线索记录 |
| 线索培育 | 营销 | 邮件、内容触达 | MQL |
| 商机创建 | 销售 | 需求确认、商机建档 | 商机记录 |
| 报价提案 | 销售 | 报价生成、审批 | 报价单 |
| 成交 | 销售 | 合同签署 | 订单 |
| 回款 | 财务 | 发票、收款 | 收款记录 |

### 4.2 关键场景

| 场景 | 触发条件 | 处理流程 | 注意事项 |
|------|----------|----------|----------|
| 线索分配 | 新线索创建 | 评分 → 规则匹配 → 分配 | 避免线索积压 |
| 线索评分 | 行为触发 | 加分/减分 → 阈值判断 → MQL/SQL | 定期校准评分规则 |
| 报价审批 | 折扣超限 | 提交审批 → 审批人审批 → 通知结果 | 明确审批权限 |
| 工单升级 | SLA 即将违约 | 自动升级 → 通知主管 → 优先处理 | 设置合理升级规则 |

## 六、最佳实践

### 6.1 实施建议

| 建议 | 原因 | 适用场景 |
|------|------|----------|
| 必填字段最小化 | 降低录入阻力，提高采纳率 | 初期上线 |
| 数据验证规则 | 保证数据质量 | 全阶段 |
| 重复检测机制 | 避免重复跟进 | 全阶段 |
| 移动端支持 | 随时随地访问 | 销售团队 |
| 与日常工具集成 | 降低切换成本 | 全阶段 |

### 6.2 常见问题

| 问题 | 解决方案 | 预防措施 |
|------|----------|----------|
| 数据质量差 | 数据清洗、验证规则、培训 | 建立数据治理机制 |
| 用户采纳低 | 简化流程、培训、激励 | 用户体验优先设计 |
| 流程过于复杂 | 流程简化、分阶段实施 | MVP 思维，逐步完善 |
| 系统响应慢 | 优化查询、归档历史数据 | 定期性能评估 |
