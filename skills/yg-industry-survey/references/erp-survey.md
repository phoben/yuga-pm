# ERP 行业知识库

## 一、行业概述

### 1.1 定义与范围

ERP（Enterprise Resource Planning，企业资源计划）是整合企业核心业务流程的管理系统，实现财务、人力、供应链、制造、采购等模块的一体化管理。

**覆盖范围**：
- 财务会计（Finance & Accounting）
- 人力资源（Human Resources）
- 供应链管理（Supply Chain Management）
- 制造管理（Manufacturing）
- 项目管理（Project Management）

### 1.2 核心特征

| 特征 | 说明 |
|------|------|
| 业务集成 | 财务、业务数据一体化 |
| 流程标准化 | 统一的业务流程规范 |
| 数据一致性 | 单一数据源，消除信息孤岛 |
| 实时可见 | 业务状态实时可查 |

### 1.3 市场格局

- **主要玩家**：SAP S/4HANA, Oracle Cloud ERP, Microsoft Dynamics 365, NetSuite, Infor, Odoo
- **市场趋势**：云化转型、AI 增强、行业化解决方案

## 二、核心模块

### 2.1 财务会计模块（Finance）

**功能说明**：
- **总账（GL）**：科目表、日记账、期间结账、财务报表、多公司合并
- **应付账款（AP）**：供应商管理、发票处理、三单匹配、付款处理
- **应收账款（AR）**：客户管理、发票生成、信用管理、收款核销
- **固定资产**：资产购置、折旧计算、资产转移、资产处置

**核心流程 - 记录到报告（R2R）**：
```
业务交易 → 凭证录入 → 过账 → 期间结账 → 对账 → 财务报表
```

**关键指标**：
| 指标名称 | 计算方式 | 说明 |
|----------|----------|------|
| 应付周转天数 (DPO) | 平均应付账款 / 日采购额 | 衡量付款效率 |
| 应收周转天数 (DSO) | 平均应收账款 / 日销售额 | 衡量收款效率 |
| 现金转换周期 | DSO + 库存周转天数 - DPO | 衡量资金效率 |

### 2.2 人力资源模块（HR）

**功能说明**：
- **核心人事**：员工档案、组织架构、岗位管理、员工生命周期
- **薪酬管理**：薪资计算、税务、扣款、发薪
- **考勤管理**：工时记录、加班计算、请假管理、排班
- **福利管理**：福利计划、参保、变更
- **招聘管理**：职位需求、应聘追踪、入职办理

**核心流程 - 招聘到退休（H2R）**：
```
需求提出 → 招聘 → 入职 → 在职发展 → 离职/退休
```

**关键指标**：
| 指标名称 | 计算方式 | 说明 |
|----------|----------|------|
| 流动率 | 离职人数 / 平均人数 × 100% | 衡量员工稳定性 |
| 招聘周期 | 职位发布到入职的平均天数 | 衡量招聘效率 |
| 人均招聘成本 | 招聘费用 / 录用人数 | 衡量招聘成本 |
| 缺勤率 | 缺勤天数 / 应出勤天数 × 100% | 衡量出勤情况 |

### 2.3 供应链模块（SCM）

**功能说明**：
- **采购管理**：采购申请、询价、采购订单、供应商管理、合同管理
- **库存管理**：库存追踪、盘点、库存计价、补货点、ABC 分析
- **仓库管理**：收货、上架、拣货、打包、发货、库位管理
- **订单管理**：订单录入、可用性承诺、订单履行、退货处理

**核心流程 - 采购到付款（P2P）**：
```
采购申请 → 审批 → 采购订单 → 收货 → 发票 → 三单匹配 → 付款
```

**核心流程 - 订单到收款（O2C）**：
```
销售订单 → 信用检查 → 履行 → 发货 → 开票 → 收款
```

**关键指标**：
| 指标名称 | 计算方式 | 说明 |
|----------|----------|------|
| 库存周转率 | 销售成本 / 平均库存 | 衡量库存效率 |
| 订单满足率 | 按时交付订单 / 总订单 × 100% | 衡量交付能力 |
| 完美订单率 | 无差错订单 / 总订单 × 100% | 衡量订单质量 |
| 供应商交付周期 | 下单到收货的平均天数 | 衡量供应效率 |

### 2.4 制造模块（Manufacturing）

**功能说明**：
- **物料清单（BOM）**：产品结构、用量、工程变更
- **生产订单**：排产、工单创建、车间控制、报工、完工
- **工艺路线**：工序顺序、工作中心、标准工时
- **生产计划**：主生产计划（MPS）、物料需求计划（MRP）、能力计划
- **质量管理**：检验方案、质量检查、不合格处理

**核心流程 - 计划到生产**：
```
需求预测 → MRP → 生产计划 → 生产订单 → 生产执行 → 完工入库
```

**关键指标**：
| 指标名称 | 计算方式 | 说明 |
|----------|----------|------|
| OEE | 可用率 × 性能率 × 质量率 | 衡量设备综合效率 |
| 生产周期 | 开始到完工的平均时长 | 衡量生产效率 |
| 产成率 | 合格品 / 投入总量 × 100% | 衡量生产质量 |
| 报废率 | 报废品 / 投入总量 × 100% | 衡量材料损耗 |

### 2.5 项目管理模块（Project）

**功能说明**：
- **项目核算**：项目设置、预算、工时费用、收入确认、开票
- **资源管理**：资源分配、利用率、技能匹配
- **工作分解结构（WBS）**：任务分解、进度跟踪

**核心流程**：
```
项目立项 → 预算编制 → 资源分配 → 执行追踪 → 收入确认 → 项目结项
```

**关键指标**：
| 指标名称 | 计算方式 | 说明 |
|----------|----------|------|
| 项目利润率 | (收入-成本) / 收入 × 100% | 衡量项目盈利 |
| 资源利用率 | 实际工时 / 可用工时 × 100% | 衡量资源使用 |
| 预算偏差 | (实际-预算) / 预算 × 100% | 衡量预算执行 |

## 三、数据模型

### 3.1 核心实体

| 实体名称 | 说明 | 核心字段 |
|----------|------|----------|
| Company | 公司/法人实体 | Code, Name, Currency, Calendar |
| Vendor | 供应商 | Code, Name, Terms, Bank Info |
| Customer | 客户 | Code, Name, Terms, Credit Limit |
| Material | 物料 | Code, Description, UOM, Price |
| Purchase Order | 采购订单 | Number, Vendor, Date, Lines, Status |
| Sales Order | 销售订单 | Number, Customer, Date, Lines, Status |
| Work Order | 生产订单 | Number, Material, Qty, Status, Dates |
| Journal Entry | 凭证 | Number, Date, Lines, Debit, Credit |

### 3.2 实体关系

```
Company --1:N--> Purchase Order
Company --1:N--> Sales Order
Vendor --1:N--> Purchase Order
Customer --1:N--> Sales Order
Purchase Order --1:N--> Receipt
Sales Order --1:N--> Shipment
Material --1:N--> Work Order
Work Order --1:N--> Material Transaction
Journal Entry --1:N--> Journal Line
```

## 四、业务流程

### 4.1 端到端流程

**采购到付款（P2P）**：
```
需求识别 → 采购申请 → 审批 → 询价 → 采购订单 → 收货 → 入库 → 发票 → 三单匹配 → 付款
```

**订单到收款（O2C）**：
```
订单接收 → 信用检查 → 库存可用性 → 拣货 → 发货 → 开票 → 收款
```

**记录到报告（R2R）**：
```
业务交易 → 凭证录入 → 过账 → 期间结账 → 对账 → 合并 → 财务报表
```

**计划到生产**：
```
销售预测 → MPS → MRP → 生产计划 → 生产订单 → 车间执行 → 完工 → 入库
```

### 4.2 关键场景

| 场景 | 触发条件 | 处理流程 | 注意事项 |
|------|----------|----------|----------|
| 三单匹配 | 发票录入 | PO-收货-发票比对 | 差异处理流程 |
| 信用检查 | 订单创建 | 检查信用额度 | 超额审批流程 |
| MRP 运行 | 需求变化 | 计算净需求 | 参数设置准确 |
| 期间结账 | 期末 | 检查未过账凭证、对账 | 截止时间控制 |

## 五、集成与扩展

### 5.1 常见集成

| 集成对象 | 集成方式 | 数据流向 | 说明 |
|----------|----------|----------|------|
| 银行系统 | 文件/API | 双向 | 付款文件、银行对账 |

### 5.2 集成模式

| EDI/XML | 交易伙伴集成 | 标准格式 |

## 六、最佳实践

### 6.1 实施建议

| 建议 | 原因 | 适用场景 |
|------|------|----------|
| 业务流程先行 | 系统适配流程，非流程适配系统 | 全阶段 |
| 数据清洗优先 | 垃圾进垃圾出 | 数据迁移前 |
| 分阶段实施 | 控制风险、快速见效 | 大型项目 |
| 变革管理 | 用户采纳是成功关键 | 全阶段 |
| 集成规划 | 避免信息孤岛 | 系统集成 |

### 6.2 常见问题

| 问题 | 解决方案 | 预防措施 |
|------|----------|----------|
| 数据迁移失败 | 数据清洗、分批迁移 | 提前规划、充分测试 |
| 用户抵触 | 培训、沟通、激励 | 变革管理 |
| 流程僵化 | 平衡标准化与灵活性 | 预留配置空间 |
| 集成困难 | 接口规范、iPaaS | 架构规划 |
