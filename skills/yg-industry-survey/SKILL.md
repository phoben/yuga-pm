# 行业调研技能

系统化调研特定行业，收集基本概念、业务特征、系统设计案例和最佳实践。

---

## 触发词

| 触发词 | 示例 |
|--------|------|
| `调查` | "调查 WMS 海外仓管理系统的行业特点" |
| `调研` | "调研电商行业的订单管理系统" |
| `行业分析` | "行业分析：物流仓储管理系统" |
| `了解行业` | "了解跨境电商 ERP 行业" |

## 知识库规范

### 文件命名规范

```
{行业关键词}-survey.md      # 行业调研文档
{系统类型}-design.md        # 系统设计参考
_template.md               # 知识库模板（请勿删除）
```

**命名示例**：
- `crm-survey.md` - CRM 行业知识库
- `wms-survey.md` - 仓储管理系统知识库
- `order-system-design.md` - 订单系统设计参考

### 文档结构规范

每个知识库文档必须包含以下章节：

| 章节 | 必填 | 说明 |
|------|------|------|
| Frontmatter | ✓ | 元数据（name, description, keywords, version, updated） |
| 一、行业概述 | ✓ | 定义、特征、市场格局 |
| 二、核心模块 | ✓ | 功能模块、核心流程、关键指标 |
| 三、数据模型 | ○ | 核心实体、实体关系 |
| 四、业务流程 | ✓ | 端到端流程、关键场景 |
| 五、集成与扩展 | ○ | 常见集成、扩展点 |
| 六、最佳实践 | ✓ | 实施建议、常见问题、调研问题清单 |
| 七、参考资料 | ✓ | 引用来源 |
| 变更记录 | ✓ | 版本变更历史 |

> ✓ 必填 ○ 选填

### Frontmatter 规范

```yaml
---
name: {行业/系统名称} Domain Knowledge
description: {一句话描述，50字以内}
keywords: [关键词1, 关键词2, 关键词3]
version: 1.0
updated: YYYY-MM-DD
---
```

### 内容编写规范

| 规范 | 说明 |
|------|------|
| 语言 | 中英文均可，保持一致 |
| 表格 | 优先使用表格呈现结构化信息 |
| 流程 | 使用 `→` 或 mermaid 图表示 |
| 代码 | 代码块标注语言类型 |
| 链接 | 参考资料必须包含可访问的链接 |

### 新增知识库流程

1. 复制 `_template.md` 模板
2. 重命名为 `{行业关键词}-survey.md`
3. 按规范填写内容
4. 更新本文件的「知识库引用」表格

## 详细流程

### 1. 意图解析

解析用户请求，提取关键信息：

| 要素 | 说明 | 示例 |
|------|------|------|
| 行业领域 | 目标行业 | WMS、电商、物流、金融 |
| 系统类型 | 具体系统 | 海外仓管理系统、订单系统 |
| 调研方向 | 关注重点 | 业务流程、技术架构、最佳实践 |
| 输出格式 | 报告形式 | 概要报告、详细报告、对比分析 |

**输出确认**：

```
调研意图确认：
- 行业领域：{行业}
- 系统类型：{系统}
- 调研方向：{方向}
- 输出格式：{格式}

是否正确？确认后开始调研。
```

### 2. 知识库检索

检查 `references/` 目录中是否有相关行业知识：

**检索步骤**：

1. 扫描 `references/` 目录下的所有 `.md` 文件
2. 根据文件名和内容匹配相关度
3. 读取匹配文档内容
4. 评估知识覆盖度

**覆盖度评估**：

| 覆盖度 | 说明 | 后续动作 |
|--------|------|----------|
| 高 (>70%) | 现有知识足够 | 直接生成报告 |
| 中 (30-70%) | 部分覆盖 | 补充网络搜索 |
| 低 (<30%) | 知识不足 | 主要依赖网络搜索 |

### 3. 信息收集

当知识库覆盖不足时，进行网络信息收集：

#### 3.1 网络搜索策略

使用 WebSearch 工具进行多维度搜索：

```
搜索维度：
1. 行业概述："{行业} 概述 定义 特点"
2. 业务流程："{行业} 业务流程 核心流程"
3. 系统架构："{系统} 系统架构 技术方案"
4. 最佳实践："{行业} 最佳实践 案例"
5. 行业报告："{行业} 行业报告 趋势分析"
```

#### 3.2 内容获取与解析

使用 WebFetch 获取网页详细内容：

- 优先获取权威来源（行业报告、技术博客、官方文档）
- 提取关键信息，过滤广告和无关内容
- 记录信息来源用于引用

#### 3.3 信息质量评估

| 维度 | 评估标准 |
|------|----------|
| 权威性 | 来源可信度、作者背景 |
| 时效性 | 信息发布时间、是否过时 |
| 完整性 | 信息覆盖范围、深度 |
| 相关性 | 与调研主题的匹配度 |

### 4. 报告生成

按标准结构生成调研报告：
