# 列出所有项目

## 使用方式

```
/yg-list
```

## 功能说明

显示当前工作空间中所有项目的状态概览，包括进行中的项目和已归档的项目。

## 执行流程

1. **检查项目目录**
   - 检查 `.yg-pm/` 目录是否存在
   - 不存在则提示用户先执行 `/yg-new` 创建项目

2. **读取进行中的项目**
   - 遍历 `.yg-pm/projects/` 目录
   - 读取每个项目的 `project.json`
   - 提取项目名称、阶段、更新时间

3. **读取已归档的项目**
   - 遍历 `.yg-pm/archive/` 目录（如存在）
   - 读取每个归档项目的 `project.json`

4. **格式化输出**
   - 以表格形式展示项目列表
   - 按更新时间排序（最近更新在前）

## 输出格式

### 进行中的项目

| 项目名称 | 阶段 | 文档数 | 最后更新 |
|---------|------|-------|---------|
| WMS优化需求 | writing | 2 | 2024-01-20 |
| 用户中心改版 | reviewing | 3 | 2024-01-18 |
| 订单系统重构 | collecting | 1 | 2024-01-15 |

### 已归档的项目

| 项目名称 | 归档时间 | 文档数 |
|---------|---------|-------|
| 旧版后台优化 | 2023-12-01 | 4 |

## 阶段状态说明

| 状态 | 说明 | 建议操作 |
|------|------|---------|
| `draft` | 项目已创建，尚未开始 | 开始需求收集 |
| `collecting` | 需求收集中 | 继续头脑风暴或提取需求 |
| `writing` | 文档编写中 | 完成文档编写 |
| `reviewing` | 评审中 | 进行需求审查 |
| `final` | 评审通过，文档定稿 | 进行原型设计或归档 |
| `on_hold` | 项目暂停 | 使用 `/yg-resume` 恢复 |
| `archived` | 已归档 | - |

## 空项目处理

如果没有项目：

```
📂 项目列表

暂无项目

使用 /yg-new <项目名> 创建第一个项目
```

## 下一步建议

- 使用 `/yg-status <项目名>` 查看项目详情
- 使用 `/yg-new <项目名>` 创建新项目
