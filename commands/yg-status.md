# 查看项目状态

## 使用方式

```
/yg-status [项目名]
```

**参数说明**：
- `项目名`：可选参数，指定要查看的项目名称
- 若不指定，则从上下文自动推断当前项目

**示例**：
```
/yg-status WMS优化需求
/yg-status
```

## 功能说明

显示指定项目的详细状态信息，包括项目元数据、已生成文档、时间线和下一步建议。

## 执行流程

1. **确定目标项目**
   - 如果指定了项目名，直接使用
   - 如果未指定，尝试从上下文推断：
     - 检查最近操作的项目
     - 检查当前工作目录
     - 检查最近修改的项目文件

2. **读取项目数据**
   - 检查 `.yg-pm/projects/{项目名}/` 目录
   - 读取 `project.json` 文件
   - 如果项目不存在，提示用户并显示可用项目列表

3. **收集详细信息**
   - 读取项目元数据
   - 遍历 `documents/` 目录获取文档列表
   - 遍历 `drafts/` 目录获取草稿列表
   - 遍历 `prototypes/` 目录获取原型列表

4. **生成状态报告**
   - 格式化输出项目详情
   - 根据当前阶段生成下一步建议

## 阶段流转图

根据当前阶段显示进度条：

```
[当前阶段: writing]

  ✓         ✓         ●         ○         ○
draft ──► collecting ──► writing ──► reviewing ──► final
```

## 各阶段建议

| 当前阶段 | 下一步建议 |
|---------|-----------|
| `draft` | 使用 `/yg-brainstorming` 开始需求收集 |
| `collecting` | 使用 `/yg-requirement-extraction` 提取规范需求 |
| `writing` | 完成文档编写，使用 `/yg-doc` 创建新文档 |
| `reviewing` | 使用 `/yg-requirement-reviewer` 进行审查 |
| `final` | 使用 `/yg-prototyping` 创建原型，或 `/yg-archive` 归档 |
| `on_hold` | 使用 `/yg-resume` 恢复项目 |

## 错误处理

**项目不存在**：
```
❌ 未找到项目 "XXX"

可用项目:
• WMS优化需求
• 用户中心改版

使用 /yg-status <项目名> 查看指定项目
```

**无法推断项目**：
```
❌ 无法确定目标项目

请指定项目名: /yg-status <项目名>

可用项目:
• WMS优化需求
• 用户中心改版
```
