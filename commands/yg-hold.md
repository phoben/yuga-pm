# 暂停项目

## 使用方式

```
/yg-hold <项目名>
```

**示例**：
```
/yg-hold WMS优化需求
```

## 功能说明

将指定项目标记为暂停状态，保存当前阶段以便后续恢复。暂停的项目不会出现在活跃项目列表的顶部。

## 执行流程

1. **验证项目存在**
   - 检查 `.yg-pm/projects/{项目名}/` 目录
   - 读取 `project.json` 文件
   - 如果项目不存在，提示错误

2. **检查当前状态**
   - 如果已经是 `on_hold` 状态，提示用户
   - 如果是 `archived` 状态，提示用户无法暂停已归档项目

3. **保存当前状态**
   - 记录当前阶段到 `previous_stage` 字段
   - 更新 `stage` 为 `on_hold`
   - 更新 `updated_at` 时间戳

4. **更新项目文件**
   - 写入更新后的 `project.json`

## project.json 变更

```json
{
  "id": "WMS-2024-001",
  "name": "WMS优化需求",
  "stage": "on_hold",
  "previous_stage": "writing",
  "updated_at": "2024-01-21T10:30:00Z",
  ...
}
```

**新增字段**：
- `previous_stage`：保存暂停前的阶段，用于恢复

## 状态流转

```
                    ┌─────────────┐
                    │             │
draft ──► ... ──► [writing] ──►   on_hold
                    │             │
                    └─────────────┘
                    保存 previous_stage
```

## 输出示例

**成功暂停**：
```
⏸ 项目已暂停

项目名称: WMS优化需求
原阶段:   writing
暂停时间: 2024-01-21 10:30

恢复命令: /yg-resume WMS优化需求
```

**项目已暂停**：
```
⚠ 项目已处于暂停状态

项目名称: WMS优化需求
暂停时间: 2024-01-20 15:00

恢复命令: /yg-resume WMS优化需求
```

**项目已归档**：
```
❌ 无法暂停已归档的项目

项目名称: WMS优化需求
当前状态: archived

归档项目无法暂停，如需恢复请使用 /yg-unarchive（如可用）
```

## 错误处理

**项目不存在**：
```
❌ 未找到项目 "XXX"

使用 /yg-list 查看所有项目
```

## 使用场景

- 项目优先级降低，暂时搁置
- 等待外部资源或依赖
- 临时转向其他紧急项目
- 需要重新评估项目方向

## 下一步建议

- 使用 `/yg-list` 查看其他项目
- 使用 `/yg-resume <项目名>` 恢复项目
