# 恢复项目

## 使用方式

```
/yg-resume <项目名>
```

**示例**：
```
/yg-resume WMS优化需求
```

## 功能说明

将暂停的项目恢复到之前的阶段，继续项目工作。

## 执行流程

1. **验证项目存在**
   - 检查 `.yg-pm/projects/{项目名}/` 目录
   - 读取 `project.json` 文件
   - 如果项目不存在，提示错误

2. **检查当前状态**
   - 如果项目状态不是 `on_hold`，提示用户
   - 检查是否存在 `previous_stage` 字段

3. **恢复之前状态**
   - 从 `previous_stage` 字段读取原阶段
   - 更新 `stage` 为原阶段
   - 移除 `previous_stage` 字段
   - 更新 `updated_at` 时间戳

4. **更新项目文件**
   - 写入更新后的 `project.json`

## project.json 变更

**恢复前**：
```json
{
  "stage": "on_hold",
  "previous_stage": "writing",
  "updated_at": "2024-01-21T10:30:00Z",
  ...
}
```

**恢复后**：
```json
{
  "stage": "writing",
  "updated_at": "2024-01-22T09:00:00Z",
  ...
}
```

## 状态流转

```
                    ┌─────────────┐
                    │             │
writing ──► on_hold ──► [writing]
                    │             │
                    └─────────────┘
                    恢复到 previous_stage
```

## 输出示例

**成功恢复**：
```
▶ 项目已恢复

项目名称:   WMS优化需求
恢复阶段:   writing
暂停时长:   2 天
恢复时间:   2024-01-23 09:00

下一步建议:
• 继续文档编写
• 使用 /yg-status 查看详细进度
```

**项目未暂停**：
```
⚠ 项目未处于暂停状态

项目名称: WMS优化需求
当前阶段: writing

项目正常运行中，无需恢复
```

**缺少原阶段记录**：
```
⚠ 无法确定原阶段

项目名称: WMS优化需求
当前状态: on_hold

原阶段记录丢失，建议:
1. 使用 /yg-status 查看项目文档
2. 根据文档完成情况手动设置阶段
```

## 错误处理

**项目不存在**：
```
❌ 未找到项目 "XXX"

使用 /yg-list 查看所有项目
```

**归档项目**：
```
❌ 无法恢复已归档的项目

项目名称: WMS优化需求
当前状态: archived

归档项目需要先从 archive/ 移回 projects/
```

## 恢复后建议

根据恢复的阶段，提供相应的下一步建议：

| 恢复阶段 | 建议 |
|---------|------|
| `draft` | 开始需求收集 |
| `collecting` | 继续需求提取 |
| `writing` | 继续文档编写 |
| `reviewing` | 进行需求审查 |
| `final` | 进行原型设计或归档 |

## 下一步建议

- 使用 `/yg-status <项目名>` 查看详细进度
- 根据恢复阶段继续相关工作
